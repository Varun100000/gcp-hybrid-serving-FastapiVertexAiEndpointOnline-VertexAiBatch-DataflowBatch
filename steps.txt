    - Vertex AI Endpoint > run Custom Docker Container > container contains Flask app + Gunicorn server + model.joblib
            - Vertex AI Endpoint is the managed serving platform.
            - Flask/Gunicorn is the container’s HTTP interface.
    - Why not “just use Vertex AI Endpoint” without Flask?
            - Because Vertex AI Endpoint does not magically know how to call your model.joblib. You have two choices:
                    - Option 1: Prebuilt Vertex AI prediction container -  Good when your model format matches a supported framework cleanly.- This can reduce custom code, but it is stricter about model format and request/response behavior.
                    - Option 2: Custom prediction container
                            - THIS IS WHAT WE ARE DOING
                            - USING THIS OPTION SINCE the recommendation response required custom post-processing and a structured top-k output, not just a single model score.
                            - THIS IS WHAT WE ARE DOING 
                                    - Client / Cloud Run API -> Vertex AI Endpoint -> Custom prediction containeR -> Flask /predict route -> model recommends products
